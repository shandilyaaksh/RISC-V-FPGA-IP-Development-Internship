## Chip Modelling Flow

The objective of the task follows the verification methodology shown below:

- O0 = Output generated by GCC compiled executable
- O1 = Output generated by the specification C model

### Verification Condition

```text
O0 = O1
```

This ensures that the implementation matches the expected behavior defined by the specification model.

## Program Used

### sum_1_to_n.c

```c
#include <stdio.h>

int main()
{
    int i, sum = 0, n = 5;

    for(i = 1; i <= n; i++)
        sum = sum + i;

    printf("Sum from 1 to %d is %d\n", n, sum);

    return 0;
}
```

## Step 4: RISC-V Compilation (-O1)

Command:


## Observations

* Native GCC compilation successfully generated the expected output.
* RISC-V GCC successfully generated RV64I executable code.
* Objdump produced the assembly instructions corresponding to the source code.
* Different optimization levels affect the generated machine instructions.
* Compilation with **-O1** optimization generated **15 instructions** in the `main()` function.
* Compilation with **-Ofast** optimization generated **12 instructions** in the `main()` function.
* The reduction in instruction count demonstrates that higher optimization levels produce more efficient machine code by eliminating redundant operations and improving execution efficiency.

---

> **Key Observation**
>
> -O1 optimization generated **15 instructions** in the `main()` function.
>
> -Ofast optimization generated **12 instructions** in the `main()` function.
>
> This demonstrates the impact of compiler optimization on instruction count and execution efficiency.

## Conclusion

The RISC-V toolchain was successfully installed and verified. A sample C program was compiled using both native GCC and RISC-V GCC. The generated executable was analyzed using objdump, providing insight into the assembly instructions generated by the RISC-V compiler.

A comparison between different optimization levels was also performed. The executable generated using **-O1** optimization contained **15 instructions** in the `main()` function, whereas the executable generated using **-Ofast** optimization contained only **12 instructions**. This demonstrates how compiler optimizations can reduce instruction count and improve code efficiency while preserving the functionality of the program.

The experiment successfully demonstrated the complete flow of C program compilation, RISC-V cross-compilation, object dump generation, and assembly-level analysis.
