## Aim

To install and verify the RISC-V GCC toolchain, compile a C program using both native GCC and RISC-V GCC, and generate the corresponding RISC-V object dump.

## Chip Modelling Flow

The objective of the task follows the verification methodology shown below:

O0 = Output generated by GCC compiled executable

O1 = Output generated by the specification C model

Verification Condition:

O0 = O1

This ensures that the implementation matches the expected behavior defined by the specification model.

## Program Used

### sum_1_to_n.c

```c
#include <stdio.h>

int main()
{
    int i, sum = 0, n = 5;

    for(i = 1; i <= n; i++)
        sum = sum + i;

    printf("Sum from 1 to %d is %d\n", n, sum);

    return 0;
}
```

---

## Step 1: Program Creation

The source file was created and edited using Leafpad.

Command:

```bash
leafpad sum_1_to_n.c
```

---

## Step 2: Native GCC Compilation

The C program was compiled using the native GCC compiler.

Command:

```bash
gcc sum_1_to_n.c
```

Execution:

```bash
./a.out
```

Output:

```text
Sum from 1 to 5 is 15
```

---

## Step 3: Display Source Code

The source file contents were verified using the cat command.

Command:

```bash
cat sum_1_to_n.c
```

---

## Step 4: RISC-V Compilation (-O1)

The program was cross-compiled using the RISC-V GCC compiler.

Command:

```bash
riscv64-unknown-elf-gcc -O1 -mabi=lp64 -march=rv64i -o sum_1_to_n.o sum_1_to_n.c
```

Explanation:

* O1 : Optimization Level 1
* mabi=lp64 : 64-bit ABI
* march=rv64i : RV64I instruction set architecture

---

## Step 5: Verify Generated Binary

The generated RISC-V executable file was verified.

Command:

```bash
ls -ltr sum_1_to_n.o
```

---

## Step 6: Generate RISC-V Object Dump

Command:

```bash
riscv64-unknown-elf-objdump -d sum_1_to_n.o
```

The generated assembly instructions were analyzed.

---

## Step 7: View Objdump Using Less

Command:

```bash
riscv64-unknown-elf-objdump -d sum_1_to_n.o | less
```

This enabled easier navigation through the generated assembly code.

---

## Step 8: Compilation Using -Ofast Optimization

The same program was compiled using a higher optimization level.

Command:

```bash
riscv64-unknown-elf-gcc -Ofast -mabi=lp64 -march=rv64i -o sum_1_to_n.o sum_1_to_n.c
```

---

## Step 9: Generate Optimized Object Dump

Command:

```bash
riscv64-unknown-elf-objdump -d sum_1_to_n.o | less
```

The optimized assembly code was analyzed and compared with the previous output.

---

## Observations

* Native GCC compilation successfully generated the expected output.
* RISC-V GCC successfully generated RV64I executable code.
* Objdump produced the assembly instructions corresponding to the source code.
* Different optimization levels affect the generated machine instructions.
* Compilation with **-O1** optimization generated **15 instructions** in the `main()` function for n=100.
* Compilation with **-Ofast** optimization generated **12 instructions** in the `main()` function for n=100.
* The reduction in instruction count demonstrates that higher optimization levels produce more efficient machine code by eliminating redundant operations and improving execution efficiency.

---

## Conclusion

The RISC-V toolchain was successfully installed and verified. A sample C program was compiled using both native GCC and RISC-V GCC. The generated executable was analyzed using objdump, providing insight into the assembly instructions generated by the RISC-V compiler.

A comparison between different optimization levels was also performed. The executable generated using **-O1** optimization contained **15 instructions** in the `main()` function, whereas the executable generated using **-Ofast** optimization contained only **12 instructions** for n=100. This demonstrates how compiler optimizations can reduce instruction count and improve code efficiency while preserving the functionality of the program.

The experiment successfully demonstrated the complete flow of C program compilation, RISC-V cross-compilation, object dump generation, and assembly-level analysis.
